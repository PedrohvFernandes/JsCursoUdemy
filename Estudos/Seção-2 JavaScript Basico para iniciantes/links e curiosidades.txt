Sobre o js: Não é recomendado juntar js no html ou ate mesmo o css no html, pois o arquivo tende a ficar enorme mais do que o esperado
mas existe o framework React que unifica o HTML com JS, mas de uma maneira elegante

Sobre front-end: a parte frontal que o usuario ve, a gente esta falando de uma pagina que é carregada,  renderizada e executada no computador client,
da pessoa que esta vendo aquela pagina, nesse caso estamos trabalhando na parte do front-end

temos o back-end: que é um servidor que vai servir as paginas e dados

Node vs chrome: mesmo que o node e o chrome tendo a mesma engine executando JavaScript eles não tem a mesma coisa e não se comportam de maneira igual

Se eu executar o metodo alert('olá mundo') pelo node(code runner) ele da um erro pelo fato de não ter uma janel/window igual no chrome ou outro navegador, o node fala
que o alert não esta definido, que ele não sabe executar essa função

Então são dois ambientes diferentes, e eles geralmente eles não tem as mesmas coisas, quando a gente trabalha no navegador a gente manipula o DOM que são
os elementos da pagina: div, p, section etc, que é o front-end do navegador

DOM -> Modelo de Objeto de Documentos

Agora com base de dados, rotas da aplicação, etc isso ae ja é o back-end o node
